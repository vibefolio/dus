## 추천 작업 순서 (코워크 중심)

### 1단계: 디자인 기반 잡기 (코워크 강력 추천)
**왜 먼저?** 디자인 없이 코드를 짜면 나중에 다 뜯어고침

- **홈페이지 디자인 프리뷰** → HTML/React로 실시간 미리보기 가능
- **컬러/폰트/레이아웃 가이드** → 일관된 디자인 시스템 구축
- **각 페이지 와이어프레임** → 구조 확정 후 코딩

### 2단계: 공통 레이아웃 (코워크 → 클로드코드)
- **네비게이션 바 + 푸터 ERB** → 코워크에서 작성
- **application.html.erb 업데이트** → 클로드코드에서 적용/테스트

### 3단계: 메인 페이지 구현 (코워크 → 클로드코드)
- **home.html.erb 풀 디자인** → 코워크에서 Tailwind 기반 작성
- **portfolio.html.erb** → 코워크에서 작성
- **contact.html.erb (견적 폼)** → 코워크에서 작성
- **서버 테스트** → 클로드코드에서 확인

### 4단계: DB 설계 (코워크)
- **ERD 다이어그램** → 코워크에서 시각화
- **마이그레이션 파일** → 코워크에서 작성, 클로드코드에서 실행

### 5단계: 어드민/백엔드 (클로드코드 중심)
- 모델 생성, 컨트롤러 로직 → 클로드코드
- 어드민 뷰 템플릿 → 코워크

## 코워크 vs 클로드코드 역할 분담

| 작업 | 코워크 | 클로드코드 |
|------|--------|------------|
| 디자인 프리뷰/목업 | ⭐ 최적 | - |
| ERB/HTML 템플릿 작성 | ⭐ 최적 | 적용/테스트 |
| 문서 작성 (기획서 등) | ⭐ 최적 | - |
| Tailwind 스타일링 | ⭐ 최적 | - |
| Stimulus JS | 작성 가능 | 테스트 |
| 마이그레이션/모델 | 파일 작성 | ⭐ 실행/테스트 |
| gem install/서버 실행 | - | ⭐ 최적 |
| DB 마이그레이션 실행 | - | ⭐ 최적 |
| Git 관리 | - | ⭐ 최적 |
