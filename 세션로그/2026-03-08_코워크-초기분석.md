# 디어스 홈페이지 코워크 초기 분석

**날짜:** 2026년 3월 8일 (일)
**세션 유형:** 프로젝트 현황 파악 및 코워크 가능 작업 정리

---

## 프로젝트 현황

### 기술 스택
- **프레임워크:** Rails 8.1.1
- **CSS:** Tailwind CSS
- **프론트엔드:** Hotwire (Turbo + Stimulus), Importmap
- **에셋 파이프라인:** Propshaft
- **DB:** SQLite3
- **배포:** Kamal + Docker

### 현재 구조
- **공개 페이지 3개:** home(메인), portfolio(포트폴리오), contact(연락/견적)
- **어드민 네임스페이스:** routes에 정의됨 (portfolios, quotes 관리)
- **컨트롤러:** PagesController만 존재 (빈 액션)
- **모델:** 아직 없음 (migration, schema 없음)
- **뷰:** 모든 페이지가 스캐폴드 기본 상태 (실제 디자인 미적용)
- **어드민 컨트롤러:** 아직 미생성

### 요약
프로젝트가 초기 셋업 단계. 라우팅과 기본 구조는 잡혀 있으나, 실제 UI/UX 디자인, 모델, 어드민 기능은 모두 구현 필요.

---

## 코워크에서 진행 가능한 작업 목록

### 1. 디자인 & UI 관련
- 페이지 디자인 목업 (HTML/React 프리뷰)
- Tailwind 기반 컴포넌트 디자인 프로토타이핑
- 반응형 레이아웃 시안 제작
- 컬러 팔레트, 타이포그래피 가이드 문서

### 2. 문서 작성
- 프로젝트 기획서 / 요구사항 정의서 (docx)
- API 설계 문서
- DB 설계 문서 (ERD 다이어그램)
- 페이지별 와이어프레임 명세서
- 배포 가이드 문서

### 3. 코드 작업 (파일 생성/편집)
- ERB 뷰 템플릿 작성 (home, portfolio, contact)
- 레이아웃 파일에 네비게이션/푸터 추가
- Tailwind 스타일링 작업
- Stimulus 컨트롤러 작성
- 모델 마이그레이션 파일 생성
- 시드 데이터 작성

### 4. 에셋 & 리소스
- SVG 아이콘/로고 제작
- 이미지 최적화
- favicon 생성

### 5. 프로젝트 관리
- 작업 진행 로그 관리
- 코드 리뷰 및 개선 제안

---

## 참고사항
- 코워크는 파일 생성/편집, 문서 작성, 디자인 프리뷰에 강점
- 서버 실행, gem install, DB 마이그레이션 등은 클로드코드(안티그래비티)에서 진행 권장
- 코워크에서 작성한 코드/파일은 프로젝트 폴더에 바로 저장되므로 클로드코드와 연계 가능
