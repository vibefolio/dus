Log: 모바일*최적화*및*템플릿*프리뷰\_개선
Date: 2025-12-26 08:12:57
Key Goal: 모바일 환경에서의 사용자 경험 개선(네비게이션, 문의 페이지) 및 디자인 템플릿 프리뷰 UI 통일, 데이터 소실 방지 조치.

📝 상세 작업 일지 (Chronological)

1. 템플릿 이미지 및 데이터 보존 (Data Persistence)
   상황: 배포 또는 시드 실행 시마다 사용자가 업로드한 이미지나 수정된 데이터가 초기화되는 문제(데이터 날라감) 발생.
   해결:
   db/seeds.rb: `DesignTemplate.destroy_all` 구문을 제거하여 기존 데이터가 있을 경우 유지되도록 로직 변경.
   기술적 포인트: 시드 파일의 멱등성(Idempotency) 보다는 데이터 보존을 우선순위로 변경.

2. 모바일 네비게이션 개선 (Mobile Navigation)
   상황: 모바일 뷰포트에서 상단 네비게이션 메뉴가 PC처럼 나열되어 레이아웃이 깨짐.
   해결:
   app/views/layouts/application.html.erb: Tailwind CSS의 `md:hidden` 클래스를 활용하여 데스크탑 메뉴를 숨기고, 햄버거 버튼과 드롭다운 형태의 슬라이드 메뉴 구현. 바닐라 JS로 토글 기능 추가.

3. 디자인 템플릿 프리뷰 UI 통일 (Template Preview Modal)
   상황: 디자인 템플릿 목록 페이지의 프리뷰 모달이 구형 스타일이며, 메인 페이지의 미리보기와 경험이 일치하지 않음.
   해결:
   app/views/design_templates/index.html.erb: 메인 페이지에서 사용 중인 'Live Mockup Modal' (다크 테마, PC/Mobile 디바이스 토글 기능 포함) 코드를 이식 및 적용.
   기술적 포인트: Iframe 제어 및 모달 애니메이션 로직 통일.

4. 문의 페이지 모바일 경험 개선 (Contact Page Mobile UX)
   상황: 모바일에서 템플릿을 'PICK'하여 문의 페이지로 이동했을 때, 화면을 가득 채우는 Iframe 미리보기가 사용자 경험을 저해함.
   해결:
   app/views/pages/contact.html.erb: 모바일 뷰(`lg:hidden`)에서는 Iframe 미리보기를 숨기고, 대신 선택한 템플릿의 썸네일과 제목을 보여주는 'Selected Template Summary' 카드를 상단에 배치. '미리보기 새창으로 열기' 링크 제공.
   기술적 포인트: Responsive Design (반응형 웹) 적용으로 디바이스별 최적화된 레이아웃 제공.
